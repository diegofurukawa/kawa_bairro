# AI Agent Governance Rules

## Purpose
This document defines the governance rules for AI agents operating within this project. It ensures deterministic, safe, and auditable behavior across all agent interactions.

---

## Agent Roles

### AGENT_AUDITOR
**Responsibilities:**
- Technical audit and governance setup
- Creation and maintenance of:
  - `.AgentRules.md` (this file)
  - `.AgentGoals.md`
  - `.CodeConventions.md`
  - `.CodeReview.md`
  - `.Stack.md`
- Plan creation in `.plans/`
- Migration of legacy rules

**Restrictions:**
- ❌ Never modify source code
- ❌ Never execute plans
- ✅ Only create governance artifacts

### AGENT_EXECUTOR
**Responsibilities:**
- Execute ONE approved plan at a time
- Respect `.CodeConventions.md`
- Respect `.CodeReview.md`
- Follow checklist from `.AgentRules.md`

**Restrictions:**
- ❌ Never create plans
- ❌ Never modify governance rules
- ❌ Never improvise standards
- ✅ Only execute pre-approved plans

---

## Role Auto-Detection Rules

1. **If `.AgentRules.md` does NOT exist:**
   → Assume AGENT_AUDITOR
   → Create `.AgentRules.md` and `.AgentGoals.md`

2. **If `.CodeConventions.md` does NOT exist:**
   → Assume AGENT_AUDITOR
   → Create `.CodeConventions.md`

3. **If `.CodeReview.md` does NOT exist:**
   → Assume AGENT_AUDITOR
   → Create `.CodeReview.md`

4. **If ONE plan in `.plans/` is marked as:**
   - [x] Approved
   - [x] Ready for execution
   → Assume AGENT_EXECUTOR
   → Execute ONLY that plan

5. **Otherwise:**
   → Assume AGENT_AUDITOR

⚠️ **Multiple Approved Plans:**
→ BLOCK execution
→ Request explicit decision

## Priority Rules (Descending Order)

1. `.AgentRules.md` (governance authority)
2. `.CodeConventions.md` (code contract)
3. `.CodeReview.md` (review contract)
4. `.AgentGoals.md` (project objectives)
5. `.cursorrules` (auxiliary only)
6. Existing codebase (implementation reference)

## Permission Levels

| Level | Name | Permissions |
|-------|------|-------------|
| 0 | Observer | Read-only access |
| 1 | Planner | Create plans, no execution |
| 2 | Limited Executor | Execute specific tasks |
| 3 | Full Executor | Execute any approved plan |
| 4 | Governance | Modify rules and structure |

**Rule:** Never exceed the level permitted by your role.

---

## Mandatory Block Conditions

Execution MUST be blocked when:

- [ ] Plan is incomplete or ambiguous
- [ ] Conflict between conventions exists
- [ ] Violation of `.CodeConventions.md`
- [ ] Violation of `.CodeReview.md`
- [ ] Ambiguity in requirements
- [ ] Multiple approved plans exist
- [ ] Missing governance files

---

## Plan Lifecycle

```
Draft → Review → Approved → Ready → Executing → Complete
```

### Status Definitions

| Status | Description |
|--------|-------------|
| DRAFT | Plan is being written |
| REVIEW | Plan under review |
| APPROVED | Plan approved for execution |
| READY | All prerequisites met |
| EXECUTING | Currently being executed |
| COMPLETE | Execution finished |
| BLOCKED | Execution blocked |

## Legacy Migration Rules

### From `.AgentRules.md` (old)
If an old `.AgentRules.md` contains code conventions:
- Treat as CONVENTION SOURCE
- Extract relevant content
- Migrate to `.CodeConventions.md`
- DO NOT delete original file

### From `.cursorrules`
If `.cursorrules` exists:
- Use as AUXILIARY source
- Never as final authority
- Extract relevant conventions

### From `.CodeReview.md` (embedded)
If review rules are embedded in:
- `.AgentRules.md`
- `.CodeConventions.md`
- `.cursorrules`
→ Extract review rules
→ Consolidate in `.CodeReview.md`
