## TypeScript Conventions

### Type Definitions
```typescript
// ✅ ALWAYS use explicit typing
interface UserProps {
  id: string;
  name: string;
  email: string;
  createdAt: Date;
}

// ✅ Prefer interfaces for objects
interface ComponentProps {
  user: UserProps;
  onEdit: (id: string) => void;
  className?: string;
}

// ✅ Use generics when necessary
type ApiResponse<T> = {
  data: T;
  success: boolean;
  message?: string;
}
```

### Prohibited Patterns
```typescript
// ❌ NEVER use any, as any, or forced casting
const data = response as any; // ❌ FORBIDDEN
```

### Client Components (When Necessary)
```typescript
// ✅ Client Component - only when interactivity is required
'use client';

import { useState, useCallback } from 'react';

interface InteractiveComponentProps {
  initialValue: string;
  onSubmit: (value: string) => Promise<void>;
}

export default function InteractiveComponent({ 
  initialValue, 
  onSubmit 
}: InteractiveComponentProps) {
  const [value, setValue] = useState(initialValue);
  
  const handleSubmit = useCallback(async () => {
    await onSubmit(value);
  }, [value, onSubmit]);
  
  return (
    <form onSubmit={handleSubmit}>
      {/* component content */}
    </form>
  );
}
```

## Custom Hooks Conventions

### Hook Structure
```typescript
// hooks/useMORequests.ts
import { useState, useEffect, useCallback } from 'react';
import type { MORequest, MORequestFilters } from '@/types';

interface UseMORequestsReturn {
  requests: MORequest[];
  loading: boolean;
  error: string | null;
  refetch: () => Promise<void>;
  updateRequest: (id: string, data: Partial<MORequest>) => Promise<void>;
}

export function useMORequests(filters?: MORequestFilters): UseMORequestsReturn {
  const [requests, setRequests] = useState<MORequest[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);
  
  const fetchRequests = useCallback(async () => {
    try {
      setLoading(true);
      setError(null);
      const data = await moRequestsService.getAll(filters);
      setRequests(data);
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Erro desconhecido');
    } finally {
      setLoading(false);
    }
  }, [filters]);
  
  const updateRequest = useCallback(async (id: string, data: Partial<MORequest>) => {
    try {
      await moRequestsService.update(id, data);
      await fetchRequests();
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Erro ao atualizar');
    }
  }, [fetchRequests]);
  
  useEffect(() => {
    fetchRequests();
  }, [fetchRequests]);
  
  return {
    requests,
    loading,
    error,
    refetch: fetchRequests,
    updateRequest,
  };
}
```

## API Route Conventions

### Route Handler Structure
```typescript
// app/api/mo-requests/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { auth } from '@/lib/auth';
import { rateLimit } from '@/lib/rate-limit';

export async function GET(request: NextRequest) {
  try {
    // Authentication
    const user = await auth.getUser(request);
    if (!user) {
      return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
    }
    
    // Rate limiting
    const rateLimitResult = await rateLimit(request);
    if (!rateLimitResult.success) {
      return NextResponse.json({ error: 'Too many requests' }, { status: 429 });
    }
    
    // Parameter validation
    const url = new URL(request.url);
    const filters = GetMORequestsSchema.parse({
      page: url.searchParams.get('page'),
      limit: url.searchParams.get('limit'),
    });
    
    const data = await moRequestsService.getAll(filters);
    
    return NextResponse.json({ data, success: true });
  } catch (error) {
    console.error('API Error:', error);
    return NextResponse.json(
      { error: 'Internal Server Error' }, 
      { status: 500 }
    );
  }
}
```

- [ ] `any` or `as any` types

- [ ] Untyped components
- [ ] Direct state mutation
- [ ] Inline CSS (except specific cases)

- [ ] Zod validation for data
- [ ] Proper error handling
- [ ] Tests for critical logic
- [ ] Documentation for complex components

## Avisos Component Patterns

### Tipo-based Rendering
Components that handle Aviso items should support both `Aviso` and `Publi` types:

```typescript
// AvisoCarouselCard supports different layouts based on tipo
const isAviso = aviso.tipo === 'Aviso'

// Publi: Image on top, title at bottom (advertising style)
// Aviso: Title on top, description in place of image, URL as link
```

### Dual View System
The `/avisos` page supports two distinct views via URL parameter `?tipo=`:

**Avisos View (`?tipo=aviso`):**
- Shows notification/message style cards
- Unread indicator (blue dot)
- Message icon
- Compact layout
- Title: "Mural de Avisos"

**Publi View (`?tipo=publi`):**
- Shows advertising card style
- Carousel at top (Publi items only)
- Larger cards with more detail
- Title: "Espaço de Publicidades"

### Auto-set Tipo on Creation
When creating new items, the tipo is automatically set based on the current view:
```typescript
const dataWithTipo = {
  ...data,
  tipo: isAvisoView ? 'Aviso' : 'Publi'
}
```
